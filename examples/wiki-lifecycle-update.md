# Example: Wiki Lifecycle Update

## User request

请根据新证据更新这个项目 wiki，并只补充必要的生命周期说明，避免写成重流程系统。

## Expected shape

- identifies the impacted page or knowledge object
- explains what changed and which evidence caused the update
- updates lifecycle markers only where they help maintenance clarity
- uses lightweight fields such as `review_status`, `last_reviewed`, `retention_class`, `consolidation_status`, or `supersedes` when justified
- keeps the update incremental instead of rewriting unrelated pages

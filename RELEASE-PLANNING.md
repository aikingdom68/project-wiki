# Release Planning Notes

## Positioning

project-wiki is strongest when positioned as:
- a wiki-first project knowledge skill
- a local-first evidence-backed explanation and evaluation layer
- a source-governed knowledge policy for future products and platforms

## Current strongest differentiators
- source-priority guided explanation
- source-guided explanation style reuse
- wiki-first rather than generic RAG-first
- project-assist outputs rather than freeform chat answers
- local-first with explicit conflict and fallback handling

## Recommended next-version priorities
1. keep source policy and output contracts stable
2. strengthen examples and golden cases
3. refine install/doctor and installation UX
4. improve page templates and audit loops
5. keep the project-knowledge identity stronger than the teaching-platform identity

# Evals Rubric

Use this rubric when reviewing Project Wiki outputs.

## Core checks

1. **Grounding**
   - Are important claims tied to local evidence?

2. **Structure**
   - Does the output match the expected task shape?

3. **Transparency**
   - Are facts, synthesis, supplements, conflicts, and gaps kept distinct?

4. **Source policy compliance**
   - If a preferred source exists, is it actually prioritized?

5. **Non-generic usefulness**
   - Does the output help real project work rather than just sounding polished?
